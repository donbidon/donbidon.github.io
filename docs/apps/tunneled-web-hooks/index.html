Tunneled web-hooks

Run tunneling service and register temporary web-hooks for workstation having no white IP by one command php bin/run.php /path/to/config.php.

  • IImplemented tunneling services: ngrok ;
  • Implemented web-hooks connectors: Telegram ;
  • Implemented bots: Windbag.

You can add your own tunneling services, register and handle your own web-hooks.
See "bin/run.php".

"hosts" file:

# Optional, just for testing
127.0.0.1 localhost.ngrok.io

nginx "io.ngrok.conf":

server {
    listen   127.0.0.1:80;
    server_name ~^(.*)\.ngrok\.io;

    root /path/to/www;
    ...
}

"config.php" file:

; <?php die; __halt_compiler();

[core.log.Stream.E_ERROR]
stream = "php://stderr"
source[] = "*"

[core.log.Stream.E_WARNING_NOTICE]
stream = "php://stdout"
source[] = "*"

[app.service]
; Set full class name (including namespace) to use own service.
class  = "ngrok"
;;;
; Put path to "ngrok.exe" here:
;;;
command = "/path/to/ngrok http 80"
;;;
; Delay after starting service.
; 3 seconds enough to start service and connect their servers at my place.
;;;
delay   = 3 ; In seconds
status  = "http://localhost:4040/status" ; Default ngrok status page

[app.webHook.Telegram.Windbag]
; Full class name (including namespace) to use own web-hook, not set by default
; class        = "\\own\\namespace\\WebHook"
;;;
; See project documentation.
;;;
url.register = "https://api.telegram.org/bot%s/setWebhook?url=%s/telegram/bot.windbag.php"
;;;
; Put your Telegram token here:
;;;
token        = "%TOKEN%"

; [app.webHook.Telegram.myBot]
; class = "\\MyNamespace\\MyBot"
; ....

[app.CLI.Console.Windbag]
prompt = "windbag> "
